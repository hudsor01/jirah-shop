# jirah-shop

A full-stack e-commerce platform built with Next.js 16, Supabase, and Stripe. Features a product catalog with variants, shopping cart, Stripe Checkout payments, admin dashboard, blog, product reviews, and coupon management.

## Tech Stack

- **Framework:** Next.js 16.1.6 (App Router, Server Components, `use cache`)
- **UI:** React 19, TailwindCSS v4, shadcn/ui
- **Database & Auth:** Supabase (PostgreSQL, Auth, Storage)
- **Payments:** Stripe (Checkout, Products, Coupons)
- **Monitoring:** Sentry (error tracking, performance)
- **Testing:** Vitest (unit/integration), Playwright (E2E)
- **Language:** TypeScript (strict mode)
- **Package Manager:** bun

## Prerequisites

- [Node.js](https://nodejs.org/) 20+
- [bun](https://bun.sh/) (package manager)
- [Supabase](https://supabase.com/) account and project
- [Stripe](https://stripe.com/) account
- [Stripe CLI](https://stripe.com/docs/stripe-cli) (for local webhook testing)

## Getting Started

### 1. Clone and Install

```bash
git clone <repo-url>
cd jirah-shop
bun install
```

### 2. Environment Variables

Create a `.env.local` file in the project root with the following required variables:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=your-supabase-publishable-key
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key

# Stripe
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Site
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

Optional (development only):

```env
ALLOW_DEV_AUTH=true  # Enables dev sign-in bypass
```

**Where to find these values:**
- **Supabase:** Project Settings > API in the [Supabase Dashboard](https://supabase.com/dashboard)
- **Stripe:** Developers > API keys in the [Stripe Dashboard](https://dashboard.stripe.com/apikeys)
- **Webhook Secret:** Generated by `stripe listen` (see Stripe Webhook Testing below)

### 3. Database Setup

Database migrations are located in `supabase/migrations/`. Apply them using the Supabase CLI:

```bash
# Install Supabase CLI if needed
brew install supabase/tap/supabase

# Link to your project
supabase link --project-ref your-project-ref

# Apply migrations
supabase db push
```

### 4. Run Development Server

```bash
bun dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Stripe Webhook Testing

The webhook handler at `app/api/webhooks/stripe/route.ts` processes `checkout.session.completed` events for order creation and stock decrement.

### Local Testing with Stripe CLI

```bash
# Install Stripe CLI
brew install stripe/stripe-cli/stripe

# Login to your Stripe account
stripe login

# Forward webhook events to your local server
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

The CLI will output a webhook signing secret (starts with `whsec_`). Copy this value and set it as `STRIPE_WEBHOOK_SECRET` in your `.env.local` file.

### Trigger Test Events

```bash
# Trigger a complete checkout flow
stripe trigger checkout.session.completed

# Trigger with specific data
stripe trigger checkout.session.completed --add checkout_session:metadata[user_id]=test-user
```

### Webhook Events Handled

| Event | Action |
|-------|--------|
| `checkout.session.completed` | Creates order, decrements stock (atomic RPC), increments coupon usage |

## Deployment

### Vercel (Recommended)

1. Push repository to GitHub
2. Import project in the [Vercel Dashboard](https://vercel.com/new)
3. Set all environment variables from `.env.local`
4. Update `NEXT_PUBLIC_SITE_URL` to your production URL
5. Configure Stripe webhook for production:
   - Go to Stripe Dashboard > Developers > Webhooks
   - Add endpoint: `https://your-domain.com/api/webhooks/stripe`
   - Select event: `checkout.session.completed`
   - Copy the signing secret to `STRIPE_WEBHOOK_SECRET`
6. Deploy

Vercel auto-detects bun from the lockfile. No additional configuration needed.

### Environment Notes

- `NEXT_PUBLIC_SITE_URL` must match your deployed domain exactly
- Use separate Stripe API keys for development vs production
- Webhook secrets are different per environment (local CLI vs Stripe Dashboard)
