```typescript
export type ActionResult<T = void> =
  | { success: true; data: T }
  | { success: false; error: string };
```

<!-- Pattern for useActionState consumers -->
```typescript
const [state, formAction, isPending] = useActionState(
  signInWithEmail,
  null  // initial state: ActionResult<void> | null
);
// Display error: state && !state.success ? state.error : null
```

<!-- Pattern for mutation consumers -->
```typescript
const result = await createProduct(formData);
if (result.success) {
  toast.success("Created");
  router.push(`/admin/products/${result.data}`);  // data is product ID
} else {
  toast.error(result.error);
}
```

<!-- Pattern for read consumers in server components -->
```typescript
const result = await getAdminProducts(options);
if (!result.success) {
  return <ErrorMessage message={result.error} />;
}
const { data, total, page, pageSize } = result.data;
```

  **cached-queries.ts** update:
  The cached queries currently wrap action functions. Since actions now return ActionResult, cached queries need to unwrap:

  ```typescript
  export async function cachedGetProducts(options?: {...}) {
    "use cache";
    cacheTag("products");
    const result = await getProducts(options);
    if (!result.success) throw new Error(result.error);
    return result.data;
  }
  ```

  This ensures storefront pages that use cached queries get raw data (not ActionResult wrapper).
  Storefront pages importing from cached-queries do NOT need changes.
