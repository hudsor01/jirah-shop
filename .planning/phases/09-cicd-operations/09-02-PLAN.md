  <action>
Create `app/api/health/route.ts` as a Next.js Route Handler:

```typescript
import { NextResponse } from "next/server"

export const dynamic = "force-dynamic"

export async function GET() {
  return NextResponse.json({
    status: "ok",
    version: process.env.VERCEL_GIT_COMMIT_SHA || "dev",
    timestamp: new Date().toISOString(),
  })
}
```

Key decisions:
- `dynamic = "force-dynamic"` ensures the response is never cached (timestamp must be fresh)
- `VERCEL_GIT_COMMIT_SHA` is auto-set by Vercel — returns "dev" in local/CI environments
- No auth required — this is a public endpoint for uptime monitors
- Simple GET-only endpoint — no POST, PUT, DELETE

Matches current local bun version. The CI workflow reads this file via `oven-sh/setup-bun` `bun-version-file`.

**Update `package.json`:**
Add `engines` field at the top level (after `"private": true`):

```json
"engines": {
  "node": ">=22.0.0",
  "bun": ">=1.3.0"
}
```

This warns developers if they run incompatible versions. The range (`>=22.0.0`) allows newer patch versions while ensuring minimum compatibility.

Also add an `analyze` script to the `scripts` section (for bundle analysis in Plan 03):
```json
"analyze": "ANALYZE=true bun run build"
```
