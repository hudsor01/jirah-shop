- Use subqueries for customer_profiles and products counts.
- `SECURITY DEFINER` (admin-only, called behind `requireAdmin()` guard).
- `RETURNS json`, `LANGUAGE plpgsql`.
- Use `coalesce(sum(...), 0)` for revenue to handle zero orders.

**00011_sales_analytics_rpc.sql** -- `get_sales_analytics(p_days integer)`:
- Default parameter: `p_days integer default 30`.
- `RETURNS TABLE(date date, order_count bigint, revenue numeric)`.
- Filters orders where status in ('paid', 'shipped', 'delivered') and `created_at >= (current_date - p_days)`.
- Groups by `date_trunc('day', created_at)::date`.
- Orders by date ascending.
- `SECURITY DEFINER`, `LANGUAGE plpgsql`.
- NOTE: This RPC returns only dates with data. JS code fills date gaps (existing pattern preserved).

Include descriptive header comments in each file matching the style of `00009_decrement_stock_rpc.sql`.
