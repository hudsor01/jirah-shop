<objective>
Add server-side pagination to all storefront listing endpoints and exclude the content column from blog listings.

Purpose: Storefront products, blog listing, product reviews, and account order history currently load ALL rows with no pagination. Blog listing also fetches the full `content` column (potentially large HTML) when only title/excerpt are displayed. All endpoints need `.range()` pagination with `{ count: 'exact' }`, and blog needs an explicit column list.

Output: Updated action functions with pagination support, updated page components to read page from searchParams and render pagination controls, and a shared storefront pagination component.
</objective>
