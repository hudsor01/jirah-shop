Add server-side pagination to storefront products and blog listing, exclude blog content column, and create shared pagination component.

Purpose: Storefront products and blog listing currently load ALL rows with no pagination. **Component: `PaginationControls`**
- Props: `{ currentPage: number, totalPages: number, baseUrl: string, searchParams?: Record<string, string> }`
- Uses `Link` from `next/link` for navigation (server component compatible).
- Renders: Previous | page numbers | Next using shadcn `Pagination`, `PaginationContent`, `PaginationItem`, `PaginationLink`, `PaginationPrevious`, `PaginationNext`, `PaginationEllipsis`.
- Shows at most 5 page numbers with ellipsis for large page counts.
- Preserves existing searchParams (category, search, sort, tag) when navigating pages.
- Returns null if totalPages <= 1 (no pagination needed).
- Builds href by appending `?page=N` to baseUrl, preserving other params.

This component is used by shop, blog, and account pages.
