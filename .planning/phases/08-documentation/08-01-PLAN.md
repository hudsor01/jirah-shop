## jirah-shop

One-paragraph description: A full-stack e-commerce platform built with Next.js 16, Supabase, and Stripe. Features product catalog, shopping cart, Stripe Checkout payments, admin dashboard, blog, reviews, and coupon management.

## Tech Stack
- Next.js 16.1.6 (App Router, Server Components, `use cache`)
- React 19
- TypeScript
- TailwindCSS v4
- shadcn/ui components
- Supabase (PostgreSQL database, Auth, Storage)

- Sentry (error tracking)
- Vitest + Playwright (testing)

## Prerequisites
- Node.js 20+
- bun (package manager)
- Supabase account and project
- Stripe account

## Getting Started

### 1. Clone and Install
```bash
git clone <repo-url>
cd jirah-shop
bun install
```

### 2. Environment Variables
Create a `.env.local` file with these required variables (reference lib/env.ts for the exact names):
- `NEXT_PUBLIC_SUPABASE_URL` - Supabase project URL
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` - Supabase publishable/anon key
- `SUPABASE_SERVICE_ROLE_KEY` - Supabase service role key (admin operations)
- `STRIPE_SECRET_KEY` - Stripe secret key
- `STRIPE_WEBHOOK_SECRET` - Stripe webhook signing secret
- `NEXT_PUBLIC_SITE_URL` - Site URL (e.g., http://localhost:3000)

Optional:
- `ALLOW_DEV_AUTH` - Set to "true" to enable dev sign-in (development only)

### 3. Database Setup
Document that Supabase migrations are in `supabase/migrations/` and need to be applied to the Supabase project. Reference the Supabase CLI for running migrations.

### 4. Run Development Server
```bash
bun dev
```
Open http://localhost:3000

## Stripe Webhook Testing

For local development, use the Stripe CLI to forward webhook events:

```bash
# Install Stripe CLI (if not already)
brew install stripe/stripe-cli/stripe

# Login to your Stripe account
stripe login

# Forward webhooks to local server
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

The CLI will output a webhook signing secret (starts with `whsec_`). Set this as `STRIPE_WEBHOOK_SECRET` in `.env.local`.

Trigger test events:
```bash
stripe trigger checkout.session.completed
```

The webhook handler is at `app/api/webhooks/stripe/route.ts` and processes `checkout.session.completed` events for order creation and stock decrement.

## Project Structure

Brief overview of key directories:
- `app/` - Next.js App Router pages and API routes
- `actions/` - Server actions (all return `ActionResult<T>`)
- `queries/` - Data access layer (Supabase queries)
- `lib/` - Shared utilities (env, auth, Stripe, sanitization, etc.)
- `components/` - React components (UI primitives and feature components)
- `hooks/` - Custom React hooks
- `providers/` - Context providers (cart, theme)
- `tests/` - Test suites (unit, integration, E2E)
- `supabase/` - Database migrations

6. Deploy

Note: The project uses `bun` as its package manager. Vercel auto-detects this from the lockfile.
