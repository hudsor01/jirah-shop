  <action>
1. In `next.config.ts`, add `experimental: { dynamicIO: true }` to the `nextConfig` object (before the `output` field is fine).

2. Create `lib/cached-queries.ts` — this file must NOT have a `"use server"` directive. It contains cached wrapper functions using the `"use cache"` directive:

```typescript
import "server-only";
import { cacheTag } from "next/cache";
import { getShopSettings } from "@/actions/settings";
import { getProducts, getFeaturedProducts } from "@/actions/products";
import { getBlogPosts } from "@/actions/blog";
```

For each cached function:
- Use the `"use cache"` directive at function level
- Call `cacheTag()` with the appropriate tag name
- Re-export the same return type as the original function

Cached functions to create:
- `cachedGetShopSettings()` — tag: `"shop-settings"`. Calls `getShopSettings()`.
- `cachedGetProducts(options)` — tag: `"products"`. Calls `getProducts(options)`. Accept same options parameter.
- `cachedGetFeaturedProducts()` — tag: `"products"`. Calls `getFeaturedProducts()`.
- `cachedGetBlogPosts(options)` — tag: `"blog"`. Calls `getBlogPosts(options)`. Accept same options parameter.

Pattern for each:
```typescript
export async function cachedGetShopSettings() {
  "use cache";
  cacheTag("shop-settings");
  return getShopSettings();
}
```

NOTE: The `"use cache"` directive goes inside the function body as the first statement, similar to `"use server"`. `cacheTag()` is called after the directive within the function body.

**Part A: Add revalidateTag to mutations**

Import `revalidateTag` from `"next/cache"` in each mutation file. Add `revalidateTag()` calls alongside existing `revalidatePath()` calls:

1. `actions/settings.ts` — in `updateShopSettings()`: add `revalidateTag("shop-settings")` alongside existing revalidatePath calls.

2. `actions/admin-products.ts` — in `createProduct()`, `updateProduct()`, `deleteProduct()`: add `revalidateTag("products")` alongside existing revalidatePath calls.

3. `actions/blog.ts` — in `createBlogPost()`, `updateBlogPost()`, `deleteBlogPost()`: add `revalidateTag("blog")` alongside existing revalidatePath calls.

Keep existing `revalidatePath()` calls — they handle admin page revalidation. `revalidateTag()` handles the `"use cache"` storefront caches.

**Part B: Update storefront pages to use cached queries**

Update these storefront pages to import from `@/lib/cached-queries` instead of directly from actions:

Check each page first — only update if the page directly imports the function. Keep the same parameters/usage pattern.
