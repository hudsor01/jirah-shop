**Part B: Admin Supabase client singleton in `lib/supabase/admin.ts`**

Convert from factory function to lazy singleton. The admin client is stateless (no session/auth cookies) so it's safe to reuse:

```typescript
import { createClient, type SupabaseClient } from "@supabase/supabase-js";
import { env } from "@/lib/env";

let adminClient: SupabaseClient | null = null;

export function createAdminClient(): SupabaseClient {
  if (!adminClient) {
    adminClient = createClient(
      env.NEXT_PUBLIC_SUPABASE_URL,
      env.SUPABASE_SERVICE_ROLE_KEY,
      {
        auth: {
          autoRefreshToken: false,
          persistSession: false,
        },
      }
    );
  }
  return adminClient;
}
```

Keep the function name `createAdminClient()` so callers don't need to change. The lazy init pattern means it's created once on first call.

**Part C: Auth caching per request in `lib/auth.ts`**

Wrap `requireAdmin` with `React.cache()` so multiple calls within a single server request reuse the result:

```typescript
import { cache } from "react";
import { createClient } from "@/lib/supabase/server";

async function _requireAdmin() {
  const supabase = await createClient();
  const { data: { user }, error } = await supabase.auth.getUser();
  if (error || !user) {
    throw new Error("Unauthorized: not authenticated");
  }
  if (user.app_metadata?.role !== "admin") {
    throw new Error("Forbidden: admin access required");
  }
  return user;
}

export const requireAdmin = cache(_requireAdmin);
```

Keep the same export name and behavior. `React.cache()` deduplicates within a single server request — if two server actions call `requireAdmin()` in the same request, only one `getUser()` call is made.

Also keep `sanitizeRedirect` and `sanitizeSearchInput` exports unchanged.

Add a `useEffect` cleanup that revokes object URLs when files change or component unmounts. Add this after the existing `onDrop` callback and before the dropzone setup:

```typescript
// Cleanup object URLs to prevent memory leaks
useEffect(() => {
  return () => {
    files.forEach((file) => {
      if (file.preview) {
        URL.revokeObjectURL(file.preview);
      }
    });
  };
  // Only cleanup on unmount — revoking during files state changes
  // would break preview display for files still in the list
  // eslint-disable-next-line react-hooks/exhaustive-deps
```

The key pattern: revoke on unmount (useEffect return), and revoke when files are explicitly removed by the user.
