   - FROM: `throw new Error(\`Variant for "${item.name}" is no longer available.\`);`
   - TO: `throw new Error("Item is currently unavailable");`

5. Collapse ALL coupon failure modes to a single generic error. Replace each of these:
   - `throw new Error(\`Coupon code "${trimmedCode}" is not valid.\`);` (line ~99)
   - `throw new Error(\`Coupon code "${trimmedCode}" has expired.\`);` (line ~103)
   - `throw new Error(\`Coupon code "${trimmedCode}" has reached its maximum uses.\`);` (line ~109)
   - `throw new Error(\`Coupon code "${trimmedCode}" requires a minimum order of $${minOrderAmount.toFixed(2)}.\`);` (line ~117)

   With this pattern — log the specific reason, then throw generic:
   ```typescript
   // For "not valid" (coupon not found):
   if (!coupon) {
     logger.warn("Coupon validation failed: not found", { code: trimmedCode });
     throw new Error("Coupon is not valid");
   }

   // For "expired":
   if (coupon.expires_at && new Date(coupon.expires_at) < new Date()) {
     logger.warn("Coupon validation failed: expired", {
       code: trimmedCode,
       expiresAt: coupon.expires_at,
     });
     throw new Error("Coupon is not valid");
   }

   // For "max uses reached":
   const maxUses = toNum(coupon.max_uses);
   const currentUses = toNum(coupon.current_uses);
   if (coupon.max_uses !== null && currentUses >= maxUses) {
     logger.warn("Coupon validation failed: max uses reached", {
       code: trimmedCode,
       maxUses,
       currentUses,
     });
     throw new Error("Coupon is not valid");
   }

   // For "min order not met":
   const minOrderAmount = coupon.min_order_amount != null ? toNum(coupon.min_order_amount) : null;
   if (minOrderAmount !== null && subtotal < minOrderAmount) {
     logger.warn("Coupon validation failed: minimum order not met", {
       code: trimmedCode,
       minOrderAmount,
       subtotal,
     });
     throw new Error("Coupon is not valid");
   }
   ```

   All four coupon error paths now throw the SAME message: "Coupon is not valid". This prevents attackers from enumerating coupon states.

6. The "Cart is empty" error at line 16 is fine as-is — not sensitive information.
