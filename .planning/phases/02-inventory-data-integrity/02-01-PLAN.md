Add a stock decrement loop:

```typescript
// ── Stock decrement (atomic RPC per line item) ──
for (const item of orderItems) {
  const { data: rowsUpdated, error: stockError } = await supabase.rpc(
    "decrement_stock",
    {
      p_product_id: item.product_id,
      p_variant_id: item.variant_id,
      p_quantity: item.quantity,
    }
  );

  if (stockError || rowsUpdated === 0) {
    logger.error(
      "Stock decrement failed — insufficient stock after checkout",
      {
        orderId: order.id,
        productId: item.product_id,
        variantId: item.variant_id || undefined,
        requestedQuantity: item.quantity,
        rpcError: stockError?.message,
      }
    );
    // Continue processing remaining items (partial fulfillment logging, not rollback)
  }
}
```

Key rules:
- Place AFTER order items insert (the order and items must exist before we decrement)
- Use a `for...of` loop (NOT `Promise.all`) -- process items sequentially so log messages are ordered
- If any item fails: log via `logger.error()` (which sends to Sentry) but DO NOT throw or break -- continue to next item
- Keep the existing idempotency check (stripe_checkout_session_id uniqueness) -- do not modify it
- Do not modify any other part of the webhook handler
